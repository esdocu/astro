November 4, 2022

Anunciando el Astro Tutorial

Por
Sarah Rainsberger
Chris Swithinbank
Dan Jutan

Esperamos que hayas disfrutado de la Astro Docs Week, ¡donde nuestros Docs core maintainers tomaron el control del teclado y de nuestra cuenta de Twitter! Cada día, publicamos un vistazo detrás de las cámaras de la vida en Team Docs:

  • nuestra iniciativa low-code de Hacktoberfest
  • nuestro translation status tracker construido a medida
  • cómo reconocemos públicamente a los contributors con Astro Badges y celebraciones personalizadas para first-time contributors

Para cerrar la semana, estamos emocionados de compartir el mayor proyecto de Docs hasta la fecha: nuestro primer tutorial de Astro.

¡Hola, mundo!

Solo hay hasta cierto punto que puedes leer; a veces, necesitas construir. En este primer tutorial project-based, construirás todas las piezas básicas de un blog de Astro desde cero.

Astro, como la web, es para todos. Fue importante para nosotros hacer este tutorial accesible para aquellos que apenas están empezando con coding. Puedes encontrar guía sobre cómo configurar tu dev environment y usar la terminal para empezar. Incluso encontrarás instrucciones para completar el proyecto enteramente en la nube. ¡Elige tu online coding environment favorito y a volar, incluso en un Chromebook o una tablet!

Así que si querías probar Astro, pero no estabas seguro de por dónde empezar, ¡echa un vistazo a nuestro tutorial Build a Blog en Docs! Y si eres un Astronauta experimentado, ahora tienes un lugar más al que puedes enviar a tus amigos Astro-curious.

¡Construye un blog hoy!

Apenas estamos empezando…

Este tutorial está recién salido del horno, y estamos buscando feedback sobre cómo mejorarlo. También tenemos nuevas ideas para temas más avanzados. ¡Asegúrate de contactar a Team Docs en Discord para decirnos qué te gustaría ver a continuación!

¡Un enorme agradecimiento a todos los involucrados en este desafiante proyecto!

Sarah Rainsberger pasó meses planificando y escribiendo, siempre defendiendo el onboarding amable del principiante de Astro, el desarrollador nuevo o el propietario de un dispositivo non-traditional.

Este proyecto requirió adiciones significativas de UI a nuestro sitio de Docs, desde componentes interactivos hasta navegación. Chris Swithinbank trabajó incansablemente para convertir palabras en una experiencia. ¡Si puedes hacer clic en algo, puedes agradecer a Chris por ello!

También tuvimos feedback temprano y útil de miembros de la comunidad. Gracias a todos los que se han unido a nosotros en calls y han contribuido fixes durante las últimas semanas. Esto, junto con la edición, testing y búsqueda de errores incansable de Dan Jutan, ayudó a enfocar y refinar nuestra dirección.

Docs Week ha terminado, pero "Docs are never done"

¿Estás interesado en involucrarte con los Docs de Astro? Echa un vistazo a nuestra Contributor's Guide para el docs repo en GitHub o encuéntranos en el canal #docs en Discord. Únete a nuestro equipo de writers, editors, translators, designers, coders, testers, builders, thinkers y dreamers. Se necesita una comunidad para producir documentación, ¡y no podemos esperar a que te unas a nosotros!
