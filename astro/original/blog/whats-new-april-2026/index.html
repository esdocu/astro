The latest community-built utilities and integrations to help you build with Astro. 🛠

  • @fujocoded/astro-authproto: Adds ATproto authentication to your Astro site. Allows your visitors to log into your website using their account on any ATproto services (like Bluesky).
  • @jdevalk/astro-seo-graph: Astro integration with SEO component, route factories, content-collection aggregator, Zod content helpers.
  • @markdy/astro: Astro island component for writing MarkdyScript animations to generate animated scenes directly in the browser.
  • @flagify/astro: An Astro integration for evaluating feature flags in pages and components, with a dev toolbar for overrides.
  • bodegacat: Astro integration for Bodega Cat e-commerce stores (Cloudflare + Stripe).
  • astro-woocommerce: Type-safe WooCommerce integration for Astro. Fetch products, categories, and manage cart state with full TypeScript support.
  • @hedystia/astro: Use @hedystia/view components within Astro.
  • @zdenekkurecka/astro-consent: Astro integration for GDPR/ePrivacy-friendly cookie consent: banner, preferences modal, runtime API, category-based consent state, strict-CSP safe, works with or without View Transitions.
  • astro-magic-move: Animated code morphing for Astro, powered by Shiki Magic Move. Build-time tokenization, zero-framework client JS, CSS-variable theming.
  • @ahmttyydn/astro: Scalar API Reference Component for Astro. Render a beautiful API reference based on an OpenAPI/Swagger document.
  • astro-pigment: An Astro 6 documentation theme with dark mode, interactive playgrounds, and SEO endpoints.
  • @casoon/astro-sitemap: Astro integration that generates sitemap.xml at build time — supports dynamic sources, pattern-based rules, i18n hreflang, serialize hook, and sitemap-index chunking.
  • @twodft/astro-icon: A community-maintained fork of astro-icon by Nate Moore, updated for Astro v6.x and Cloudflare Workers compatibility.
  • @omniaura/astro-grab: Hold a key, hover, click — grab any element’s source context for AI agents. Astro devtool.
  • @casoon/astro-crawler-policy: Policy-first crawler control for Astro — generates robots.txt and llms.txt with presets, per-bot rules, AI crawler registry, and build-time audits.
  • astro-bangumi: A Bangumi integration for Astro, similar to hexo-bilibili-bangumi.
  • astro-include-asset: Include assets on your page by writing a custom remark directive in your Markdown/MDX file.
  • @pliosoft/sites-sdk: Shared Astro components, layouts, and design tokens for Pliosoft-hosted customer sites.
  • astro-cloudflare-analytics: An Astro component to handle manual Cloudflare Analytics installation.
  • astro-research-publications: Astro integration that syncs your academic publications from Google Scholar, Semantic Scholar, OpenAlex, and/or ORCID and renders a publications page.
  • astro-llms-md: Generate llms.txt, llms-full.txt, and individual markdown files from your Astro site with this Astro integration.
  • @red-glare/astro: Astro integration for building USWDS-styled federal documentation sites with Comark.
  • @soufianeh/annotate-code: Link code lines to explanations — an Astro integration and remark plugin with hover highlighting.
  • @jessgaspardev/astro-seo: A fully-featured SEO component for Astro with Open Graph, Twitter Cards, JSON-LD and more.
  • @dotslide/framework: A modern presentation framework for Astro. Build fast, portable decks from ready-made semantic components instead of custom slide code.
  • astro-omni-compress: A drop-in alternative to sharp for Astro’s <Image> component that works wherever sharp’s native binaries fail — Docker alpine images, serverless functions, Vercel Edge, Cloudflare Workers.
  • @at-flux/astro-feature-flags: Typed Astro feature flags from JSON config. Includes environment specific rendering and pruning from elements to entire routes.
  • @twodft/astro-favicons: An all-in-one favicon and PWA assets generator for Astro projects. It automates the creation of favicons, manifest, and supports hot reloading for efficient development.
  • @tsparticles/astro: Easily create highly customizable particle, confetti and fireworks animations with ready to use components and use them as animated backgrounds.
  • @obaronai/astro-ai-readiness: AI Readiness toolkit for Astro.
  • astro-tinylytics: Astro components for Tinylytics analytics, kudos buttons, and webmentions.
  • astro-meta-engine: Powerful, type-safe metadata and JSON-LD engine for Astro.
  • astro-gtm-lite: Lightweight Google Tag Manager compatible with Astro View Transitions (client router).
  • @mintlify/astro - Syncs Mintlify-style docs content into Astro content collections and provides MDX components for rendering docs pages.
  • astro-dgmo: Astro integration to render DGMO diagrams from fenced code blocks at build time.
  • astro-opengraph - An integration to create and manage your Open Graph images, including a dev toolbar app to preview the image and the og meta tags on the page.
  • @ayco/astro-resume: Utilities for serializing data from the server for use in the client.
  • @better-translate/astro: A type-safe i18n toolkit for TypeScript.
  • astlide: An Astro-based slide presentation framework — like Slidev, but for the Astro ecosystem.
  • astro-feedkit: An Astro integration for full-content RSS, Atom, and JSON feeds.
  • astro-mdx: An Astro integration for MDX directive-to-component mapping, element overrides, and auto-imports.
  • astro-search-plugin: Framework-agnostic, type-safe search for Astro 5+ powered by Orama.
  • astro-agent-optimized: Agent-optimised output for Astro. Inspired by laravel/pao.

Plugins

  • Astro Cursor Plugin: Cursor plugin for Astro including the official Astro Docs MCP, rules with best practices and coding standards and several skills such as add integration, content collection, create component, docs lookup and migrate.
  • ESLint astro/no-prerender-export-outside-pages: An eslint-plugin-astro rule for Astro that warns when export const prerender is used outside the pages/ directory.

Skills

  • astro-agent-skills: A curated collection of agent skills that give AI assistants deep knowledge of Astro patterns, best practices, and workflows. Works with Codex, Cursor, Claude Code, and other agents that support the open skills ecosystem.

CMS/text editors

  • star-lite-docs: Starlight-style documentation theme for EmDash CMS — Astro integration + emdash plugin in one package.
  • phage-cms: Astro integration: Phage CMS (Cloudflare KV/R2, block editor, API routes).
  • nebula-cms: A streamlined, Git-based Content Management System built specifically for Astro.
  • @a83/orbiter-integration: Astro integration for Orbiter CMS — admin UI, virtual modules, injected routes.
  • clay-cms: An Astro-native, adapter-based content backend with a Payload-style config file and an admin UI built with nothing but Astro.

Content loaders

  • notro-loader: A Content loader for Astro that uses the Notion Public API.
  • @dylmye/atproto-standard-site-astro-loader: Astro loader for Standard Site documents (pckt.blog, leaflet.pub, etc.) - add your blog posts to your Astro site!
  • @fujocoded/astro-atproto-loader: Astro loaders (live and static) for reading AtProto records into Astro content collections.
  • astro-loader-s3: An Astro content loader for loading Markdown files from S3-compatible object storage (AWS S3, Cloudflare R2, DigitalOcean Spaces, and others).

Dev toolbar apps

  • astro-click-to-ai: An Astro Dev Toolbar app that captures clicked elements as a JSON list with annotations for AI coding assistants.

Starlight

  • 🌟 starlight-theme-terminal: Astro Starlight theme with retro and modern elements.
  • 🌟 starlight-skills: A Starlight plugin that turns your documentation into AI-installable skills.
  • 🌟 starlight-llm-actions: A Starlight plugin that adds a Page Actions dropdown to every doc page with copy/view as Markdown, save as PDF, open with <provider>
  • 🌟 starlight-recipes: A Starlight plugin to create a recipe website: cooking mode, printable design, optimised SEO through Structured Data (JSON-LD) and rating system.
